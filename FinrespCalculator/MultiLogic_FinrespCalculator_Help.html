1. Индикатор расчёта

При нажатии Рассчитать вверху страницы появляется синяя полоса «Идёт расчёт»: после заголовка — полоса-ползунок (заполнение слева направо), справа — текст этапа; процент в конце строки этапа (… — 45%):

  • подбор фьючерсов MOEX (если есть фьючерсы) — начало диапазона загрузки;
  • загрузка цен MOEX / кэш IndexedDB — 0–33% по числу загруженных инструментов;
  • расчёт FINRESP — 33–66% по свечам (240/480 свечей в тексте); без Stopper этот этап идёт до 99%;
  • Stopper портфеля (если SL/TP включены) — 66–99%, текст Stopper портфеля · N/M свечей;
  • во время расчёта — не выше 99%; 100% и закрытие полосы только после полного завершения (графики и результат).
  • справа в полосе — кнопка Остановить: прерывает загрузку MOEX и расчёт; показываются FINRESP и графики до последней досчитанной свечи (правая граница окна сдвигается). При остановке во время загрузки — по уже загруженным инструментам.

На время расчёта тема страницы слегка меняется, все поля и кнопки формы заблокированы (кроме «Остановить»). После завершения блокировка снимается.

2. Запуск

Калькулятор — MultiLogic_FinrespCalculator.html + MultiLogic_FinrespCalculator.engine.js в одной папке. Для MOEX нужен http://, не file://.

  • Локально: serve-calculator.ps1 → http://127.0.0.1:8765/MultiLogic_FinrespCalculator.html
  • GitHub Pages: онлайн-версия
  • Перенос: скопируйте папку FinrespCalculator целиком

При открытии через file:// появляется жёлтое предупреждение — свечи MOEX не загрузятся.

Настройки формы (режим кошелька, даты, объёмы, логики L1…L5, галочки, live-инструменты в «Заявка»/«Стакан», @OBT) сохраняются в localStorage браузера под ключом multilogic.finresp.config.v1 и восстанавливаются при следующем открытии. Токен T-Bank — отдельно, зашифрован.

5.1. Параметры @ (основной блок)

В синей карточке «Основной расчёт», сразу под выбором логики — поля подстановки в строку Op/Cl. Символ @ — плейсхолдер; при «Рассчитать» и «Применить» числа подставляются в SL[@SL], LinReg(@LR;K=@K), Strict(@Strict) и т.д.

ПолеНазначение
@SL — стоп, ×ATRКратность ATR для stop-loss позиции; 0 — выкл. Не абсолютная цена
@TP — тейк, ×ATRTake-profit позиции; кратность ATR
Длина ATR, свечейПериод ATR в барах (свечах) — общий для @SL/@TP, @SmaCorridor и @K
@SmaCorridor — SMA, ×ATRШирина коридора ±K×ATR вокруг SMA (логики SMA-эталон); не в свечах
@LR — LinReg, свечейДлина линейной регрессии в барах (свечах), не в ATR. Кнопка «Оптимизировать»
@K — LinReg, ×ATRШирина канала: по умолчанию 2×ATR (не свечи, не абсолютная цена). UT/UCT: LinReg(@LR;K=@K). Кнопка «Оптимизировать»
@Strict — строгостьStrict(@Strict) в начале строки; 1…5. Кнопка «Оптимизировать»
@Reverse — реверсГалочка: инверсия покупка ↔ продажа для всех логик; не в строке Op/Cl. Кнопка «Оптимизировать» (вкл/выкл). При вкл. блок @ подсвечивается оранжевым

Серая строка-сводка под полями показывает текущие значения подстановок в логику. Кнопки «Оптимизировать» у @SL, @TP, длины ATR, @LR, @K, @Strict и @Reverse перебирают значения по FINRESP.

9.5. Используемые индикаторы

Чекбоксы SMA, ATR, Stoch, LinReg, Bollinger, VWAP, CCI, MACD, Momentum. Отключённые удаляются из Op/Cl перед расчётом. SL/TP позиции не зависят от этого списка.

Оптимизировать (индикаторы) — перебор непустых комбинаций, остаётся набор с лучшим FINRESP.

9.6. Каталог логик (TBC, UT, UCT, L5, L1…L4, SMA…)

Первая — TBC (test counter-bokovik): лонг, Stoch+MACD, контртренд-боковик (ниже SMA на вход). Далее UT, UCT, L5, L1…L4.

Текстовые поля Op/Cl/Regime — как в роботе MultiLogic. Применить — подставить @SL, @TP, @LR, @K, @Strict из основного блока и пересчитать. Маркер @OBT в строку добавляется вручную.

10.1. Графики по инструментам (calc-chart)

После «Рассчитать» (или в live-сессии при обновлении графиков) для каждого инструмента строится отдельный блок с тикером и FINRESP по этой бумаге.

Цена — свечи

  • Отображаются японские свечи (Open / High / Low / Close) по барам выбранного окна.
  • Зелёная свеча — close ≥ open; красная — close < open.
  • Данные OHLC берутся из загруженных свечей MOEX/T-Bank; при расчёте они также попадают в строки симуляции.

Масштаб и прокрутка

График интерактивный — удобно смотреть плотные участки (много баров на маленькой ширине):

ДействиеЭффект
Колёсико мыши над графикомМасштаб по времени (zoom in/out) вокруг точки под курсором
Перетаскивание (зажатая ЛКМ)Сдвиг видимого окна влево/вправо по барам
Двойной кликВернуть отображение всего периода окна

В подсказке на графике показывается, сколько баров видно из общего числа (если включён zoom).

Линии индикаторов

На графике рисуются линии индикаторов, которые используются в выбранных логиках (галочки в блоке «Логика»), а не только у «активной» логики в стеке:

  • SMA — оранжевая пунктирная; для SMA-коридора — также верхняя/нижняя граница коридора
  • LinReg — фиолетовая (центр и полосы ±Dev или ±K×ATR для BlLinK / AbRegK и т.п.)
  • Bollinger — голубая (средняя и верх/низ)
  • VWAP — зелёная пунктирная

Если индикатор отключён в «Индикаторы» или не входит ни в одну выбранную логику — линия не рисуется.

Маркеры входа и выхода

Над свечами (по high бара) отмечаются смены позиции по результату симуляции:

МаркерЗначение
△ остриём вниз, только контур (не залит)Вход в позицию (long или short) — переход с нулевой позиции на ненулевую
▲ остриём вверх, залитый тёмно-зелёныйВыход по сигналу логики (закрытие позиции)
▲ залитый красныйВыход по стоп-лоссу (SL)
▲ залитый ярко-зелёныйВыход по тейк-профиту (TP)

У SMA-логик с непрерывным набором/сбросом объёма маркеры ставятся при переходе позиции через ноль (и при срабатывании SL/TP, если включены).

Вертикальные линии и фон (как раньше)

  • SL / TP по позиции — тонкая вертикальная: SL красная, TP зелёная.
  • SL / TP портфельного стоппера — жирнее и с другим пунктиром (на всех графиках инструментов).
  • В live — пунктирные линии заявок: покупка синяя, продажа оранжевая.
  • В live при смене режима — цветной фон: зелёный — песочница (фейк), розовый — реальная торговля.

Заявка (свёрнутый блок)Свой выпадающий список Инструмент (из бумаг калькулятора), покупка/продажа, рыночная/лимитная, лоты, «Выставить заявку». При лимите и смене инструмента цена подставляется автоматически: сначала close последней свечи выбранного тикера, иначе последняя цена T-Bank (не остаётся цена прошлого инструмента). Стакан (свёрнутый блок)Отдельный список Инструмент в красной строке над таблицей (справа — глубина, время, спред; тикер в строке — не селектор). Данные T-Bank GetOrderBook, опрос ~4 с при развёрнутом блоке. Двойной клик по цене (подчёркнутая ячейка bid/ask) — развернуть «Заявку» и заполнить: инструмент стакана, лимит, цена клика, сторона, ровно 1 лот; заявка не отправляется автоматически

После «Начать торговлю» — позиции, изменённые с начала сессии (неизменённые старые скрыты). Без сессии — все открытые на счёте. Не все тикеры MOEX доступны для торговли через API T-Bank (api_trade_available_flag). При «выбрать все акции» калькулятор предпочитает листинг MOEX (TQBR), проверяет статус до заявки и пропускает недоступные бумаги, продолжая торговлю по остальным. Токен должен иметь право на торговлю (не только чтение).

При live без предварительного «Рассчитать» или при расхождении окон свечей возможно предупреждение «FINRESP пустой» / windowSkipped — свечи и опрос продолжаются, торговля не останавливается; сузьте выбор или нажмите «Рассчитать».
